# Store schema

Canonical description of the on-disk shape of `cairn.db` — the embedded
better-sqlite3 store with the `sqlite-vec` extension loaded.

The DDL that this document describes is generated by
[`src/main/db/migrations/0001-init.ts`](../../src/main/db/migrations/0001-init.ts).
The migration runner contract is documented in
[ADR 0003](../adr/0003-local-store-migrations.md).

## Location

- **File name**: `cairn.db`
- **Directory**: `resolvePaths().data` (see
  [`src/shared/paths.ts`](../../src/shared/paths.ts)):
  - Windows: `%APPDATA%\Cairn\cairn.db`
  - macOS: `~/Library/Application Support/Cairn/cairn.db`
  - Linux: `$XDG_DATA_HOME/cairn/cairn.db` (or `~/.local/share/cairn/cairn.db`)
- **Backups**: pre-migration backups land under
  `resolvePaths().data/backups/cairn.db.<timestamp>.backup` (see the
  migration runner).

## Runtime pragmas

Every connection opened via `openDatabase()` runs the following pragmas
before the migration runner:

| PRAGMA                 | Value       | Why                                                                 |
|------------------------|-------------|---------------------------------------------------------------------|
| `journal_mode`         | `WAL`       | Concurrent read/write robustness; migration runner also truncates. |
| `foreign_keys`         | `ON`        | better-sqlite3 defaults to OFF — must be enabled per connection.   |
| `user_version`         | integer     | Single source of truth for schema version; compared with `CODE_SCHEMA_VERSION`. |

## Extensions

- **sqlite-vec** is loaded via `db.loadExtension(getLoadablePath())`. The
  extension backs the `vec_items` virtual table. If the loadable extension
  is not present on the platform, `openDatabase()` throws.

## Schema descriptor

The `LocalStoreSchema` contract
([`src/contracts/local-store.contract.ts`](../../src/contracts/local-store.contract.ts))
exposes the following top-level fields:

| Field           | Type                | Meaning                                     |
|-----------------|---------------------|---------------------------------------------|
| `version`       | `number`            | Migration key (`PRAGMA user_version`).      |
| `tables?`       | `readonly string[]` | Entity table names.                         |
| `indexes?`      | `readonly string[]` | Secondary-index names.                      |
| `virtualTables?`| `readonly string[]` | Extension-backed tables (e.g. `vec_items`). |
| `migrationsDir?`| `string`            | Directory holding migration files.          |
| `backupsDir?`   | `string`            | Directory where pre-migration backups land. |

New fields are additive; the `DO NOT MODIFY EXPORTS WITHOUT A VERSIONING ADR`
banner in the contract file governs any change.

## Tables — migration 0001-init.ts

The initial migration creates 22 tables (20 regular + 1 companion
`vector_metadata` + 1 `vec0` virtual). Every regular table follows the
convention: `id TEXT PRIMARY KEY`, `created_at TEXT NOT NULL`, and
`updated_at TEXT NOT NULL` (UTC ISO-8601). Status columns are constrained
by `CHECK` clauses so the DB rejects out-of-band values even when the
caller bypasses TypeScript.

### Core entity tables

| Table              | Notes                                                                                    |
|--------------------|------------------------------------------------------------------------------------------|
| **people**         | Squad members and contributors. Indexed by `status`.                                     |
| **projects**       | PoC / squad projects. Indexed by `status`.                                               |
| **charters**       | Long-form project charter body (Markdown). FK `project_id` (CASCADE on delete).          |
| **knowledge_items**| Generic ingest surface. Indexed by `type`.                                               |
| **news_topics**    | Curated topics grouping news items.                                                      |
| **news_items**     | External news rows. FK `topic_id` (SET NULL). Indexed by `topic_id`, `published_at`.     |
| **docs**           | Long-form docs. FK `project_id` (SET NULL). Indexed by `project_id`.                     |
| **tickets**        | Local issue tracker. FKs `project_id`, `assignee_id` (both SET NULL).                    |
| **wip_signals**    | Git-derived WIP signals. Composite index (`entity_type`, `entity_id`) + status index.    |
| **updates**        | Free-form project updates. FKs `project_id` (CASCADE) and `author_id` (SET NULL).        |
| **decisions**      | Recorded decisions. FKs `project_id` and `decided_by` (both SET NULL).                   |
| **apps**           | Third-party tools tracked by the squad.                                                  |
| **meetings**       | Meeting records. `attendee_ids` is a JSON string of person ids.                          |
| **action_items**   | Actions from meetings. FKs `meeting_id` (CASCADE) and `owner_id` (SET NULL).             |
| **reports**        | Rendered reports. FK `project_id` (SET NULL).                                            |
| **feeds**          | RSS/Atom/GitHub feeds. `enabled` CHECK (0 or 1).                                         |
| **budget_ledger**  | Project budget ledger. FK `project_id` (SET NULL).                                       |
| **jobs**           | Background jobs; indexed by `status`.                                                    |
| **sync_state**     | Per-entity sync cursor. UNIQUE index on `entity_type`.                                   |
| **settings_kv**    | Key-value settings (the "settings" table). **Never used for secrets** — keychain only.   |

The status enums for the tables above are enumerated in
[domain-model.md](./domain-model.md#status-enum-reference).

### Vector storage

`sqlite-vec` cannot store arbitrary metadata columns in the `vec0` virtual
table (0.1.6 does not expose stable auxiliary/partition semantics). Cairn
therefore joins metadata via a companion table on `rowid`:

| Table              | Kind             | Purpose                                                              |
|--------------------|------------------|----------------------------------------------------------------------|
| **vector_metadata**| Regular table    | Row-level filter columns: `entity_id`, `entity_type`, timestamps. UNIQUE index on `(entity_type, entity_id)`, secondary index on `entity_type`. |
| **vec_items**      | `vec0` virtual   | Packed `float[1536]` embeddings. Dimension is baked into the DDL — bumping it requires a new migration that drops and recreates `vec_items`. |

Metadata filtering (`entity_type='project'`) happens at the DAO layer by
joining `vec_items` to `vector_metadata` on `rowid`. See
[`src/main/db/dao/vectors.ts`](../../src/main/db/dao/vectors.ts).

The `vec_items` embedding dimension is **1536** (matches OpenAI ada-002 /
`text-embedding-3-small`; a common upper bound for local embedding models).

## Migration checklist

Before you land a new migration file (e.g. `0002-*.ts`):

1. **Bump `CODE_SCHEMA_VERSION`** in
   [`src/main/db/schema.ts`](../../src/main/db/schema.ts) to match the new
   migration's `VERSION` export.
2. **Register the migration** in
   [`src/main/db/migrations/index.ts`](../../src/main/db/migrations/index.ts).
   The runner asserts the registry is `1..N` gap-free.
3. **Forward-only**: no `down()` — the runner rejects a DB that reports a
   `user_version` higher than the code's `CODE_SCHEMA_VERSION` with a
   `NewerSchemaVersionError`. See [ADR 0003](../adr/0003-local-store-migrations.md).
4. **Transactional apply**: the runner opens a single `BEGIN…COMMIT`
   around each migration's `up()`; do NOT open your own transaction.
5. **Backup first**: the runner runs `PRAGMA wal_checkpoint(TRUNCATE)` and
   copies `cairn.db` to a timestamped backup before applying any pending
   migration. First-run installs skip this because there is no file yet.
6. **Update docs**: extend this file's tables list and refresh
   [domain-model.md](./domain-model.md) with the new entities / enum states.
